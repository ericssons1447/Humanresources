# RecruitFlow AI (ATS-lite + Interview Support)

RecruitFlow AI is a full-stack, recruiter-focused ATS-lite application that supports:
- candidate document intake,
- resume/cover letter parsing,
- job description analysis,
- explainable candidate-to-job scoring,
- recruiter table review,
- export to CSV / Excel,
- and initial scaffolding for interview and recommendation workflows.

> **Important Compliance Notice**
> - This system is designed to support human decision-making.
> - Recruiters must manually review all recommendations.
> - No auto-rejections are performed.
> - Protected characteristics (race, sex, age, religion, disability, national origin) are not scored.
> - Names/photos/addresses/graduation years are not intended scoring drivers.

## Tech Stack
- Next.js 14 + TypeScript (App Router)
- Prisma ORM + SQLite
- File upload handling via Next route handlers
- PDF/DOCX/TXT extraction (`pdf-parse`, `mammoth`, plain text fallback)
- CSV + Excel-compatible export (`xlsx`)

## Local Setup

1. **Install dependencies**
```bash
npm install
```

2. **Create env file**
```bash
cp .env.example .env
```

3. **Generate Prisma client + migrate database**
```bash
npm run prisma:generate
npm run prisma:migrate -- --name init
```

4. **Seed sample data**
```bash
npm run prisma:seed
```

5. **Start app**
```bash
npm run dev
```

6. Open `http://localhost:3000`

---

## Usage Walkthrough
1. Go to **New Job** and create a posting by pasting/uploading a job description.
2. Go to **Candidate Upload** and upload resume + optional docs.
3. Visit **Candidates** for the recruiter table and exports.
4. Open a candidate to see:
   - parsed profile,
   - required vs preferred qualification breakdown,
   - explainable scorecard and evidence,
   - recommendation rationale,
   - audit trail entry.

---

## Notes on AI Integration
Current parsing/scoring is implemented as deterministic mock utilities in `lib/analysis.ts`. The architecture is intentionally modular so real AI/LLM calls can be swapped in later without changing UI or DB contracts.

---

## Known Limitations
- PDF/DOCX parsing quality depends on source document format.
- Candidate list sorting/filtering is lightweight MVP behavior.
- Interview question generation and full interviewer response workflow remain Phase 2 TODO.
- Full manual score override UI and score-change audit actions remain Phase 3 TODO.
